===========================================
AJAX API ENDPOINTS
===========================================

All return JSON: {success: true/false, message: "", data: {}}

/api/add-to-cart.php
- POST: product_id, quantity
- Validates: product exists, stock available
- Updates: $_SESSION['cart']
- Returns: cart_count, cart_total, message

/api/update-cart.php
- POST: product_id, quantity
- Validates: stock
- Updates: session
- Returns: item_subtotal, cart_subtotal, cart_total

/api/remove-from-cart.php
- POST: product_id
- Removes from session
- Returns: cart_count, cart_total

/api/apply-coupon.php
- POST: coupon_code, cart_total
- Validates: exists, active, not expired, min amount, uses left
- Calculates: discount_amount
- Stores: $_SESSION['coupon']
- Returns: discount_amount, new_total, message

/api/newsletter-signup.php
- POST: email
- Validates: format, not duplicate
- Inserts: into newsletter table
- Sends: welcome email
- Returns: success, message

/api/search.php
- GET: q (query string)
- Searches: product name, description
- Returns: array of {id, name, slug, price, image, category}
- Limit: 10 results

Security for all APIs:
- Validate and sanitize inputs
- Use prepared statements
- Check CSRF token (if implemented)
- Return proper JSON headers
- Log errors, don't expose to client

===========================================
SECURITY MEASURES
===========================================

CRITICAL security implementations:

1. SQL INJECTION PREVENTION:
   - Use PDO prepared statements ALWAYS
   - Never concatenate user input into queries

2. XSS PREVENTION:
   - Escape output: htmlspecialchars()
   - Validate input types

3. CSRF PROTECTION:
   - Generate token in forms
   - Validate on submission

4. PASSWORD SECURITY:
   - Use password_hash() (bcrypt)
   - Never store plain text passwords

5. SESSION SECURITY:
   - Regenerate session ID on login
   - Secure and HttpOnly flags
   - Session timeout

6. FILE UPLOAD SECURITY:
   - Validate file types
   - Check file size
   - Rename uploaded files
   - Store outside web root if possible

7. HTTPS:
   - Force HTTPS (redirect HTTP)
   - Secure cookies

8. INPUT VALIDATION:
   - Server-side validation always
   - Sanitize all inputs
   - Type checking
